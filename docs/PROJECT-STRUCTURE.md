# Project layout & disk usage

## Why the folder looks ~1.5–1.7 GB

Almost all size is **generated or installed** — not your source code.

| Location | Typical size | Role |
|----------|----------------|------|
| **`.next/`** | ~750–800 MB | Next.js build + **webpack cache** (` .next/cache` ~590 MB) + dev traces (` .next/dev` ~190 MB). **Safe to delete**; recreated on `npm run dev` / `npm run build`. |
| **`node_modules/`** | ~730–750 MB | App dependencies (Next, Playwright, AWS SDK, etc.). **Normal**. Remove only if you plan to run `npm ci` again. |
| **`bookmarkfs/`** | ~20–25 MB | Browser extension subproject + its own **`bookmarkfs/node_modules`** + `dist/` bundles. |
| **`legacy/`** | ~20 MB | Older/static assets and code kept for reference — **optional** to keep in the repo. |
| **Source (`app/`, `components/`, `lib/`, etc.)** | &lt; ~5 MB | Actual product code. |

**Measured total (this machine):** ~**1.58 GB** top-level sum; **`dir /s`** reported ~**1.66 GB** / **~54,800 files** (mostly under `node_modules` and `.next`).

## Recommended folder organization (mental model)

| Path | Purpose |
|------|---------|
| **`app/`** | Next.js App Router — pages, layouts, `api/` routes. |
| **`components/`** | React UI (workspace shell, landing, settings, `ui/` primitives). |
| **`lib/`** | Shared server/client logic (db, auth helpers, plans, mail, etc.). |
| **`workers/`** | Long-running processes (e.g. FFmpeg conversion worker), not Vercel serverless. |
| **`drizzle/`** | SQL migration snapshots (see repo conventions). |
| **`docs/`** | Operator and developer documentation. |
| **`tests/`** | Vitest (and related) tests. |
| **`public/`** | Static assets served as-is. |
| **`scripts/`** | Node/tsx maintenance scripts (`clean:next`, admin, migrations). |
| **`bookmarkfs/`** | Separate npm package: Webpack-built extension (`npm run extension:build` from repo root). |
| **`legacy/`** | Non-runtime legacy material — archive or drop if you do not need it. |
| **`samples/`** | Example inputs / fixtures. |
| **`.github/`** | CI and Dependabot. |

## Surface boundaries

For planning and cleanup, treat the repo as two product surfaces that happen to ship together:

- **WOX-Bin workspace surface**: `app/app`, `app/p`, `app/raw`, `app/feed*`, `app/archive`, `app/settings`, `app/admin`, most of `components/workspace`, and the paste/account services under `lib/`.
- **WOX Tools surface**: `app/tools`, conversion APIs under `app/api/convert`, worker code under `workers/convert`, and the tool/conversion helpers in `lib/tools`, `lib/convert`, `lib/pdf-extract`, and related UI under `components/tools`.

That separation is deliberate. It keeps Vercel audits, product decisions, and future refactors from mixing paste-specific concerns with utility/conversion concerns.

## What you can delete safely (reclaim space)

1. **`.next/`** — run **`npm run clean:next`** (or delete the folder). **Frees ~0.75–0.8 GB** immediately.
2. **`tsconfig.tsbuildinfo`** (repo root) — TypeScript incremental cache; **safe to delete** (~few MB).
3. **Playwright / test artifacts** (if present): `playwright-report/`, `test-results/`, `coverage/` — already in `.gitignore`; delete locally anytime.
4. **`bookmarkfs/node_modules`** — safe to delete if you are not building the extension; run `npm install` inside `bookmarkfs/` when needed again.

## What is “unnecessary” only in specific cases

| Item | When it’s unnecessary |
|------|------------------------|
| **`legacy/`** | You no longer need old site/assets for reference or migration. |
| **`bookmarkfs/dist/`** | Regenerated by `extension:build`; can omit from backups if you always rebuild. |
| **Root `node_modules/`** | Unnecessary **only** until the next `npm ci` / `npm install`. |

## What **not** to delete casually

- **`app/`**, **`components/`**, **`lib/`**, **`auth.ts`**, config files — your application.
- **`.env.local`** — secrets (never commit; listed in `.gitignore`).

## App UI theme (light / dark / system)

- Stored in **`localStorage`** as `woxbin_ui_theme` (`dark` | `light` | `system`). Legacy `woxbin_app_shell_light` is still written for compatibility.
- Applied on **every route** via the inline script + **`ThemeRootProvider`** in `app/layout.tsx` (Tailwind `dark` class + `theme-light` CSS variables).
- The workspace header (**`/app`**) has the **App** dropdown to change it; other pages follow the last saved choice.

## Scripts

- **`npm run clean:next`** — removes `.next` (see `scripts/rm-next.mjs`).
- **`scripts/disk-usage-report.ps1`** — top-level size breakdown (Windows).
- **`scripts/disk-usage-detail.ps1`** — deeper `.next` / `bookmarkfs` detail.

---

## Keeping the repo small in Git

`.gitignore` already excludes `.next`, `node_modules`, coverage, Playwright output, etc. **Do not commit** build folders or `node_modules`; clone + `npm ci` + `npm run build` restores them.
